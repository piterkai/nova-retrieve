SmartDoc 平台架构纪要（v3.2，2025-Q4）

1. 总体分层
- 接入层：Nginx + 自研 API 网关，负责鉴权、限流、灰度。
- 应用层：Go 微服务集群，约 40 个服务，按业务域划分（文档、协作、搜索、AI）。
- 数据层：MySQL 8.0（业务库，分库分表 32 片）、Redis 7（缓存与分布式锁）、Qdrant（向量库）、对象存储 OSS（文档原文）。
- 异步层：Kafka + Flink，负责审计日志、搜索索引、AI 任务编排。

2. AI 子系统
- Embedding 服务统一使用 BGE-M3，部署在 T4 GPU 集群，单实例 QPS ~ 60。
- RAG 流水线由 LangGraph 实现，支持查询改写、相关性评分、幻觉检测、Web 兜底。
- LLM 走内部网关，可路由到 DeepSeek、通义、智谱、Azure OpenAI；具体路由由策略中心下发。

3. 容量规划（2025-12）
- 日均文档新增 12 万份，总量 6.8 亿份。
- 向量库：Qdrant 集群 6 节点，单节点 32C/128G/4TB NVMe，HNSW 索引 m=32 ef_construct=256。
- 高峰 QPS：搜索 1500，AI 问答 280。

4. 稳定性指标
- 文档读取 P99 < 80ms；搜索 P99 < 200ms；AI 问答 P95 < 4s。
- 整体可用性 SLO 99.95%，目前 30 天滚动可用性 99.97%。
- 灾备：主备双活（上海 + 北京），RPO 30 秒，RTO 5 分钟。

5. 近期变更
- 2025-10：搜索召回引入 BGE-Reranker，召回质量提升约 18%（NDCG@10）。
- 2025-11：RAG 模块上线幻觉检测节点，回答错误率从 7.2% 降至 2.4%。
- 2025-12（计划）：将向量库由单租户拆为按企业租户隔离的多 collection 架构。
